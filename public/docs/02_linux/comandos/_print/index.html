Introducción a los Comandos de Linux

Los comandos en Linux son instrucciones que el usuario ejecuta en la terminal para interactuar directamente con el sistema operativo.

Linux, como sistema basado en Unix, ofrece una amplia gama de comandos que permiten realizar tareas de administración, manipulación de archivos, gestión de procesos y redes , entre otros.

Estos comandos forman el núcleo de la interacción en Linux , especialmente para usuarios avanzados, desarrolladores y administradores de sistemas.

Núcleo de Comandos Linux y Herramientas GNU

El sistema operativo Linux incluye un conjunto básico de comandos integrados en el núcleo.

Estos comandos básicos, como ls (para listar archivos), cd (para cambiar de directorio) y ps (para listar procesos en ejecución), son esenciales y consisten en una colección relativamente pequeña.

Sin embargo, muchas distribuciones Linux incluyen las utilidades GNU (como chmod, grep, awk, y sed), un conjunto extensivo de herramientas que amplían enormemente las capacidades de Linux.

Las utilidades GNU son fundamentales en casi todas las distribuciones, aportando entre 400 y 600 comandos adicionales que permiten manipular archivos, gestionar usuarios y grupos, editar textos, y mucho más.

Las herramientas GNU son esenciales en sistemas Linux modernos y forman el corazón de muchas operaciones en la terminal.

Distribuciones y Variabilidad de Comandos

La cantidad exacta de comandos disponibles en Linux puede variar significativamente según la distribución y los paquetes instalados.

Algunas distribuciones como Ubuntu, Debian y Red Hat incluyen sus propias herramientas de administración y gestores de paquetes (apt, yum, o dnf), que agregan comandos específicos.

Además, herramientas populares como Docker, Git, y Kubernetes, cada una con sus propios subcomandos y opciones, contribuyen al número total de comandos disponibles.

Un Entorno Extensible

La flexibilidad de Linux permite instalar paquetes y aplicaciones de terceros, lo que significa que el número de comandos disponibles puede extenderse aún más.

En una instalación básica de Linux, un usuario puede contar con aproximadamente 1,000 comandos, mientras que en una instalación avanzada, el número puede superar los 2,000, o incluso más, dependiendo del software instalado.

Esta extensibilidad y la modularidad del sistema Linux hacen de sus comandos una herramienta poderosa y adaptable a una gran variedad de necesidades, desde la administración de servidores hasta el desarrollo de aplicaciones y el análisis de datos.

En este tema, veremos algunos comandos, si bien, es interesante profundizar y según el objetivo del manejo del sistema, podremos seguramente indagar en nuevas funcionalidades.

¿Qué Son los Comandos en Linux?

Los comandos en Linux permiten controlar el sistema desde la interfaz de línea de comandos (CLI) en lugar de usar una interfaz gráfica.

Son instrucciones escritas en texto que, al ser introducidas en el terminal, indican al sistema exactamente qué hacer.

Esto proporciona un control preciso y detallado sobre el sistema, algo que es difícil de lograr solo con el ratón en un entorno gráfico.

Algunas características clave de los comandos en Linux:

  • Distinguen entre mayúsculas y minúsculas: Por ejemplo, ls y LS son diferentes, ya que Linux es un sistema sensible a las mayúsculas.
  • Tienen una estructura definida: Usan la sintaxis de comando -opciones argumentos, donde cada comando puede llevar opciones y argumentos específicos.
  • Combinación para tareas avanzadas: Se pueden combinar para realizar operaciones complejas mediante pipelines (|) y redirecciones (> y >>).
  • Automatización y scripting: Los comandos permiten crear scripts para automatizar tareas y ejecutar procesos en lote, lo que facilita la gestión y operación del sistema.
  • Acceso directo a recursos del sistema: Puedes gestionar el sistema de archivos, la red, la memoria y la CPU directamente desde la línea de comandos.
  • Interacción en sistemas de servidor: La CLI es la base para la administración remota de servidores Linux, permitiendo a los administradores realizar configuraciones, instalaciones y monitoreo de sistemas de forma eficiente.

Usar comandos en Linux es esencial para usuarios avanzados y administradores de sistemas .

Comandos Generales

1. ls

Lista los archivos y directorios en el directorio actual. 

1

ls -la

  • -l: lista en formato largo.
  • -a: incluye archivos ocultos.

2. cd

Cambia de directorio. 

1

cd /ruta/del/directorio

3. cp

Copia archivos o directorios. 

1

 cp archivo_origen archivo_destino

  • -r: para copiar directorios de forma recursiva.

4. mv

Mueve o renombra archivos o directorios. 

1

mv archivo_origen archivo_destino

5. rm

Elimina archivos o directorios. 

1

rm archivo

  • -r: elimina directorios de forma recursiva.
  • -f: fuerza la eliminación sin confirmación.

6. cat

Muestra el contenido de un archivo.

1

cat archivo

7. grep

Busca texto dentro de archivos. 

1

grep 'texto' archivo

8. chmod

Cambia permisos de archivos o directorios.

1

chmod 755 archivo

9. chown

Cambia el propietario de archivos o directorios. 

1

chown usuario

10. find

Busca archivos en el sistema. 

1

find /ruta -name 'archivo*'

Comandos de Redes y Comunicaciones

1. ifconfig

Muestra o configura las interfaces de red. 

1

ifconfig

2. ip

Muestra o manipula rutas, dispositivos y políticas de red. 

1

ip addr show

  • ip route: muestra la tabla de rutas.
  • ip link: muestra o configura dispositivos de red.

3. ping

Envía paquetes ICMP para verificar la conectividad con una IP o dominio. 

1

ping google.com

4. netstat

Muestra conexiones de red, tablas de enrutamiento y estadísticas.

1

netstat -tuln
  • -t: muestra conexiones TCP.
  • -u: muestra conexiones UDP.
  • -l: muestra servicios en escucha.

5. ss

Muestra información sobre sockets de red. 

1

ss -tuln

6. traceroute

Muestra la ruta que siguen los paquetes hasta un host. 

1

traceroute google.com

7. nslookup

Interroga a servidores DNS para obtener información de dominios o IPs. 

1

nslookup google.com

8. dig

Realiza consultas DNS más detalladas. 

1

dig google.com

9. ftp / sftp

Transfiere archivos entre sistemas mediante los protocolos FTP/SFTP. 

1

ftp usuario@servidor sftp usuario@servidor

10. scp

Copia archivos entre sistemas de forma segura. 

1

scp archivo usuario@servidor:/ruta/destino

11. curl

Realiza peticiones HTTP desde la terminal. 

1

curl https://ejemplo.com

12. wget

Descarga archivos de una URL. 

1

wget https://ejemplo.com/archivo.tar.gz

13. nmap

Escanea redes y puertos. 

1

nmap -sP 192.168.1.0/24

14. ssh

Conexión remota segura a otro sistema. 

1

ssh usuario@servidor

15. iptables

Configura las reglas del cortafuegos en el sistema.

1

iptables -L

Comandos de Administración de Archivos en Linux

1. Navegación y Exploración del Sistema de Archivos

  • cd : Cambia de directorio (change directory) 

    1

    cd [directorio]

    • Ejemplos: 

      1
2

      cd /usr/local #: Cambia al directorio `/usr/local`.
cd .. #: Sube un nivel en el sistema de archivos.

    • ls : Lista el contenido de un directorio (list). 

      1

      ls [opciones] [directorio]

      • Opciones útiles:
        • -l: Muestra detalles como permisos, propietario, tamaño y fecha de modificación.
        • -a: Muestra archivos y directorios ocultos.
        • -R: Lista de forma recursiva el contenido de subdirectorios.
        • -S: Ordena por tamaño.
        • -t: Ordena por tiempo de modificación.
      • Ejemplo: 
        1

        ls -l /home/user/documentos

  • pwd : Muestra la ruta del directorio actual (print working directory). 

    1

    pwd

  • tree : Muestra la estructura de directorios en forma de árbol (tree).

    • Nota: Por defecto, tree no suele estar instalado. Para instalarlo: 
      1
2

      sudo apt update
sudo apt install tree
    • Ejemplos: 
      1
2
3

      tree [directorio] #: Muestra la estructura completa del directorio.
tree -L 2 /home/user #: Muestra solo dos niveles de profundidad en el directorio /home/user.
tree -L 1 /var/log #: Muestra únicamente el primer nivel de subdirectorios en /var/log.

  • find : Busca archivos y directorios en el sistema (find).

    • Este comando es muy potente y habría que practicarlo un poco, pues puede ser de mucha ayuda en momentos concretos. 
      1

      find [ruta] [opciones] [criterio]

      • Opciones comunes:

        • -name [nombre]: Busca archivos o directorios que coincidan con el nombre especificado (sensible a mayúsculas).
        • -iname [nombre]: Igual que -name, pero insensible a mayúsculas.
        • -type [tipo]: Especifica el tipo de archivo a buscar:
          • f para archivos regulares.
          • d para directorios.
        • -mtime [+n/-n]: Busca archivos modificados hace más (+n) o menos (-n) de n días.
        • -size [tamaño]: Busca archivos de un tamaño específico, por ejemplo, +10M para archivos mayores a 10 MB.
        • -user [usuario]: Busca archivos que pertenezcan a un usuario específico.
        • -exec [comando] {} \;: Ejecuta un comando sobre cada archivo encontrado, reemplazando {} por el nombre del archivo.

      • Ejemplos: 

        1
2
3
4
5
6

        find /home/user -name "archivo.txt" #: Busca el archivo "archivo.txt" en el directorio /home/user.
find / -iname "*.pdf" #: Busca archivos PDF en el sistema de forma insensible a mayúsculas.
find /var/log -type f -size +10M #: Busca archivos en /var/log mayores a 10 MB.
find . -type f -mtime +30 #: Busca archivos modificados hace más de 30 días en el directorio actual.
find /tmp -user nombre_usuario #: Busca archivos en /tmp pertenecientes al usuario especificado.
find /home -name "*.log" -exec rm {} \; #: Busca y elimina todos los archivos .log en /home.

  • locate : Busca archivos rápidamente (locate).

    • Nota: locate no suele estar instalado en el sistema, por lo que hay que instalarlo y actualizar la base de datos para poder proceder a búsquedas rápidas: 
      1
2
3

      sudo apt update
sudo apt install locate
sudo updatedb
    • Ejemplos: 
      1
2
3

      locate index.html #: Busca archivos llamados "index.html" en el sistema.
locate -i documento.pdf #: Realiza una búsqueda insensible a mayúsculas de "documento.pdf".
locate /etc/apt/sources.list #: Encuentra la ubicación del archivo "sources.list" en el sistema.

2. Manipulación de Archivos y Directorios

  • cp : Copia archivos o directorios (copy). 

    1

    cp [opciones] /origen/ /destino/

    • Opciones útiles:
      • -r: Copia directorios de forma recursiva.
      • -i: Pide confirmación antes de sobrescribir.
      • -v: Muestra detalles de los archivos copiados.
    • Ejemplo: 
      1

      cp -r /home/user/documents /backups/ #: Copia el directorio "documents" al directorio "backups".

  • mv : Mueve o renombra archivos y directorios (move). 

    1

    mv [opciones] /origen/ /destino/

    • Ejemplos: 
      1
2

      mv folder1 folder2 #: Renombra "folder1" a "folder2".
mv /tmp/file.txt /home/user/ #: Mueve "file.txt" al directorio "/home/user/".

  • rm : Elimina archivos y directorios (remove). 

    1

    rm [opciones] /nombre/

    • Opciones útiles:
      • -r: Elimina directorios recursivamente.
      • -f: Fuerza la eliminación sin confirmación.
    • Ejemplo: 
      1

      rm -rf temp #: Elimina el directorio "temp" y su contenido sin confirmación.

  • mkdir : Crea un nuevo directorio (make directory). 

    1

    mkdir [opciones] <directorio>

    • Opciones:
      • -p: Crea directorios principales según sea necesario.
      • -v: Muestra el directorio creado.
    • Ejemplo: 
      1

      mkdir -v ~/project/code #: Crea el subdirectorio "code" dentro de "project" y muestra el resultado.

  • touch : Crea un archivo vacío o actualiza la fecha de modificación (touch). 

    1

    touch [opciones] /nombredearchivo

    • Ejemplo: 
      1

      touch /home/user/nuevoarchivo.txt #: Crea un archivo vacío llamado "nuevoarchivo.txt" en "/home/user".

  • rmdir : Elimina un directorio vacío (remove directory). 

    1

    rmdir [opciones] <directorio>

    • Opciones:
      • -v: Muestra el directorio eliminado.
    • Ejemplo: 
      1

      rmdir -v ~/project/code #: Elimina el directorio "code" dentro de "project" y muestra el resultado.

3. Visualización y Edición de Archivos

  • cat : Muestra el contenido de un archivo (concatenate). 

    1

    cat [archivo]

    • Ejemplo: 
      1

      cat documento.txt #: Muestra el contenido completo de "documento.txt".

  • less / more : Visualiza archivos largos de forma paginada (less y more). 

    1
2

    less [archivo]
more [archivo]

    • Ejemplos: 
      1
2

      less log.txt #: Muestra "log.txt" permitiendo desplazarse con las teclas.
more log.txt #: Muestra "log.txt" avanzando una pantalla a la vez.

  • head : Muestra las primeras líneas de un archivo (head). 

    1

    head [opciones] [archivo]

    • Ejemplo: 
      1

      head -n 5 notas.txt #: Muestra las primeras 5 líneas de "notas.txt".

  • tail : Muestra las últimas líneas de un archivo (tail). 

    1

    tail [opciones] [archivo]

    • Ejemplo: 
      1
2

      tail -n 10 log.txt #: Muestra las últimas 10 líneas de "log.txt".
tail -f log.txt #: Muestra en tiempo real las nuevas líneas añadidas a "log.txt".

  • nano, vim, gedit : Editores de texto en terminal. 

    1
2
3

    nano [archivo]
vim [archivo]
gedit [archivo]

    • Ejemplos: 
      1
2
3

      nano notas.txt #: Abre "notas.txt" para editar en nano.
vim config.php #: Abre "config.php" para editar en vim.
gedit texto.txt &: Abre "texto.txt" en gedit en segundo plano.

4. Análisis y Procesamiento de Archivos

  • grep : Busca texto usando patrones (global regular expression print). 

    1

    grep [opciones] pattern [archivos]

    • Ejemplo: 
      1

      grep -i "error" /var/log/syslog #: Busca "error" en syslog ignorando mayúsculas.

  • awk : Procesamiento de texto basado en patrones y acciones (Aho, Weinberger, Kernighan). 

    1

    awk 'pattern { acción }' input-file

    • Ejemplo: 
      1

      awk '/error/ {print $1}' /var/log/syslog #: Imprime el primer campo de cada línea que contenga "error".

  • sed : Editor de secuencias para filtrado y transformación de texto (stream editor). 

    1

    sed [opciones] 'commands' input-file

    • Ejemplo: 
      1

      sed 's/foo/bar/' file.txt #: Reemplaza "foo" con "bar" en "file.txt".

  • sort : Ordena líneas de archivos de texto (sort). 

    1

    sort [opciones] [archivo]

    • Ejemplo: 
      1

      sort -n datos.txt #: Ordena numéricamente el archivo "datos.txt".

  • uniq : Filtra líneas repetidas adyacentes (unique). 

    1

    uniq [opciones] [entrada]

    • Ejemplo: 
      1

      sort datos.txt | uniq #: Elimina líneas duplicadas en "datos.txt" después de ordenar.

  • diff : Compara archivos línea por línea (difference). 

    1

    diff [opciones] archivo1 archivo2

    • Ejemplo: 
      1

      diff original.txt actualizado.txt #: Muestra diferencias entre "original.txt" y "actualizado.txt".

  • wc : Imprime recuento de líneas, palabras y bytes (word count). 

    1

    wc [opciones] [archivo]

    • Ejemplo: 
      1

      wc reporte.txt #: Muestra el número de líneas, palabras y bytes de "reporte.txt".

5. Análisis de Espacio de Disco

  • du : Estima el uso del espacio de un archivo o directorio (disk usage). 

    1

    du [opciones] [ruta]

    • Opciones útiles:
      • -h: Muestra los tamaños en un formato legible para humanos (KB, MB, GB).
      • -s: Muestra solo el tamaño total de un directorio.
      • -a: Muestra el tamaño de cada archivo individual.
    • Ejemplo: 
      1

      du -sh pictures #: Muestra el tamaño total de la carpeta "pictures".

  • df : Muestra el uso de espacio en disco para cada sistema de archivos montado (disk free). 

    1

    df [opciones]

    • Opciones útiles:
      • -h: Muestra el uso de espacio en formato legible para humanos.
      • -T: Muestra el tipo de sistema de archivos.
      • -i: Muestra el uso de inodos en lugar de bloques de disco.
    • Ejemplo: 
      1

      df -h #: Muestra el uso de espacio en disco en un formato amigable.

  • lsblk : Lista los dispositivos de bloques (discos y particiones) (list block devices). 

    1

    lsblk [opciones]

    • Opciones útiles:
      • -f: Muestra el sistema de archivos de cada dispositivo.
      • -l: Muestra la lista en un formato lineal.
    • Ejemplo: 
      1

      lsblk -f #: Muestra todos los dispositivos y sus sistemas de archivos.

  • fdisk : Muestra y administra la tabla de particiones de discos (format disk). 

    1

    fdisk [opciones] [dispositivo]

    • Opciones útiles:
      • -l: Lista las particiones de todos los discos.
    • Ejemplo: 
      1

      fdisk -l #: Muestra las particiones de todos los discos disponibles.

  • ncdu : Analiza el espacio en disco con una interfaz visual (NCurses Disk Usage).

    • Nota: ncdu requiere instalación previa: 
      1
2

      sudo apt update
sudo apt install ncdu
    • Ejemplo: 
      1

      ncdu /home/user #: Muestra el uso de espacio en "/home/user" de forma interactiva.

  • btrfs : Comando específico para sistemas de archivos Btrfs (B-Tree file system). 

    1

    btrfs filesystem df [punto de montaje]

    • Ejemplo: 
      1

      btrfs filesystem df /mnt #: Muestra el uso de espacio en un sistema Btrfs montado en "/mnt".

  • lsof : Lista archivos abiertos y ayuda a identificar qué archivos están en uso y consumiendo espacio (list open files). 

    1

    lsof +D [directorio]

    • Ejemplo: 
      1

      lsof +D /var/log #: Muestra archivos abiertos en "/var/log".

Comandos Generales

1. ls

Lista los archivos y directorios en el directorio actual. 

1

ls -la

  • -l: lista en formato largo.
  • -a: incluye archivos ocultos.

2. cd

Cambia de directorio. 

1

cd /ruta/del/directorio

3. cp

Copia archivos o directorios. 

1

 cp archivo_origen archivo_destino

  • -r: para copiar directorios de forma recursiva.

4. mv

Mueve o renombra archivos o directorios. 

1

mv archivo_origen archivo_destino

5. rm

Elimina archivos o directorios. 

1

rm archivo

  • -r: elimina directorios de forma recursiva.
  • -f: fuerza la eliminación sin confirmación.

6. cat

Muestra el contenido de un archivo.

1

cat archivo

7. grep

Busca texto dentro de archivos. 

1

grep 'texto' archivo

8. chmod

Cambia permisos de archivos o directorios.

1

chmod 755 archivo

9. chown

Cambia el propietario de archivos o directorios. 

1

chown usuario

10. find

Busca archivos en el sistema. 

1

find /ruta -name 'archivo*'

Comandos de Redes y Comunicaciones

1. ifconfig

Muestra o configura las interfaces de red. 

1

ifconfig

2. ip

Muestra o manipula rutas, dispositivos y políticas de red. 

1

ip addr show

  • ip route: muestra la tabla de rutas.
  • ip link: muestra o configura dispositivos de red.

3. ping

Envía paquetes ICMP para verificar la conectividad con una IP o dominio. 

1

ping google.com

4. netstat

Muestra conexiones de red, tablas de enrutamiento y estadísticas.

1

netstat -tuln
  • -t: muestra conexiones TCP.
  • -u: muestra conexiones UDP.
  • -l: muestra servicios en escucha.

5. ss

Muestra información sobre sockets de red. 

1

ss -tuln

6. traceroute

Muestra la ruta que siguen los paquetes hasta un host. 

1

traceroute google.com

7. nslookup

Interroga a servidores DNS para obtener información de dominios o IPs. 

1

nslookup google.com

8. dig

Realiza consultas DNS más detalladas. 

1

dig google.com

9. ftp / sftp

Transfiere archivos entre sistemas mediante los protocolos FTP/SFTP. 

1

ftp usuario@servidor sftp usuario@servidor

10. scp

Copia archivos entre sistemas de forma segura. 

1

scp archivo usuario@servidor:/ruta/destino

11. curl

Realiza peticiones HTTP desde la terminal. 

1

curl https://ejemplo.com

12. wget

Descarga archivos de una URL. 

1

wget https://ejemplo.com/archivo.tar.gz

13. nmap

Escanea redes y puertos. 

1

nmap -sP 192.168.1.0/24

14. ssh

Conexión remota segura a otro sistema. 

1

ssh usuario@servidor

15. iptables

Configura las reglas del cortafuegos en el sistema.

1

iptables -L
